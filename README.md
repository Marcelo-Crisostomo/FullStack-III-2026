## 📖 Descripción de la Asignatura

La asignatura de **Desarrollo Full Stack III** tiene como objetivo principal que los estudiantes de 5to semestre aprendan a diseñar, construir y desplegar aplicaciones web robustas y escalables utilizando una arquitectura moderna orientada a **Microservicios**.

Durante el curso, aprenderemos a separar las responsabilidades del sistema en servicios independientes, gestionar la comunicación entre ellos, y construir una interfaz de usuario dinámica y responsiva que consuma estos servicios de forma segura y eficiente.

*Nota: Este repositorio y los proyectos incluidos tienen fines estrictamente académicos.*

---

## 🛠️ Tecnologías y Herramientas

De acuerdo al programa formativo (disponible en la carpeta `Info`), utilizaremos las siguientes tecnologías en el curso:

### **Backend y Microservicios**
*   **Java** (Lenguaje principal del backend).
*   **Spring Boot** (Framework para la creación ágil de microservicios).
*   **Spring Cloud** (Gestión de configuración, enrutamiento y descubrimiento: *API Gateway*, *Eureka Server*, etc.).
*   **Gestión de Base de Datos** (SQL / JPA Hibernate).

### **Frontend**
*   **React / Angular** (Creación de interfaces de usuario dinámicas SPA y manejo del DOM moderno).
*   **HTML5 & CSS3** (Maquetación y estilos).
*   **Bootstrap** (Diseño responsivo y componentes de interfaz).

### **Herramientas de Desarrollo y Testing**
*   **Git & GitHub** (Control de versiones).
*   **Postman** (Testeo de APIs REST y recolección de endpoints).
*   **Node.js / NPM** (Gestor de paquetes para el frontend).

## 🚀 Instrucciones para Estudiantes

Como estudiante, tendrás acceso 24/7 a este código. Para empezar a usarlo en tu equipo local:

1.  **Clonar el repositorio:**
    ```bash
    git clone https://github.com/TU_USUARIO/FullStack-III-2026.git
    ```
2.  **Mantenerte actualizado:**
    Regularmente el profesor subirá actualizaciones o nuevos ejemplos. Para traer los últimos cambios a tu equipo, utiliza:
    ```bash
    git pull origin main
    ```
3.  **Explorar las carpetas `EA`:**
    Dentro de cada carpeta de Experiencia de Aprendizaje, encontrarás un `README` específico (si aplica) con los pasos para levantar la base de datos, el backend y el frontend.
